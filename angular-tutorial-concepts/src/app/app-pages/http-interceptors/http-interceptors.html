HTTP Interceptor

HTTP Interceptor in Angular

An HTTP Interceptor is a special Angular service that sits between your app and the backend API.

It can inspect, modify, or handle:
Outgoing HTTP requests
Incoming HTTP responses
Errors globally
Think of it as a middleware layer for all HTTP calls.

Why do we use HTTP Interceptors? (Real-Time Use Cases)

In real projects, interceptors are used for:

  • Add Auth Token (JWT) to headers
  • Handle API errors globally
  • Show / hide loader (spinner)
  • Log API requests & responses
  • Modify request URLs or headers
  • Retry failed requests

Instead of repeating code in every service, interceptor does it once globally

How HTTP Interceptor Works (Flow)

  • Component calls API via HttpClient
  • Request goes to Interceptor
  • Interceptor modifies request (if needed)
  • Request hits backend
  • Response comes back through Interceptor
  • Component receives final response
