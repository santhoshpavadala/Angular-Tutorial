Angular Signals(16+)

Signals:

A Signal in Angular is a reactive state container that holds a value and automatically notifies Angular to update the UI whenever that value changes.

A Signal is:
A variable
That Angular tracks automatically
And re-renders the UI when it changes

What are Signals in Angular?

Signals are a new reactive state management primitive introduced in Angular 16.

    They allow you to:
  • Store reactive values
  • Automatically track dependencies
  • Update the UI when data changes
  • Avoid manual subscriptions (like RxJS subscribe())

Think of Signals as a reactive variable that automatically notifies Angular when it changes.

Why Signals Were Introduced?

Before Signals:
We used Input(), EventEmitter
RxJS BehaviorSubject, Subject
Manual subscribe()
ChangeDetectionStrategy tricks
This often caused:
Memory leaks
Unnecessary change detection cycles
Complex state logic
Signals solve this by:
Being synchronous
Automatically tracking dependencies
Triggering fine-grained change detection

1. Writable Signal

A Writable Signal is a signal whose value you can read and update manually.
It’s like a normal variable but reactive.

Use cases
Form input values
Toggle state
Counters
UI flags (loading, open/close)

2. Computed Signal

A Computed Signal is a derived value that is automatically calculated based on other signals. You cannot set it manually.

When price or quantity changes → total updates automatically

Use cases
Calculations (total price)
Filtering lists
Derived UI state

3. Effect

An Effect runs a function automatically when dependent signals change, mainly for side effects.

Important
Used for side effects, NOT for storing values Runs automatically when signals change

Use cases
API calls
Logging
Local storage updates
DOM interactions
