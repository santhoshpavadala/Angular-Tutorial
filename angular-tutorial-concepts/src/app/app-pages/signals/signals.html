Angular Signals(16+)

What are Signals in Angular?

Signals are a new reactive state management primitive introduced in Angular 16.

    They allow you to:
  • Store reactive values
  • Automatically track dependencies
  • Update the UI when data changes
  • Avoid manual subscriptions (like RxJS subscribe())

Think of Signals as a reactive variable that automatically notifies Angular when it changes.

Why Signals Were Introduced?

Before Signals:
We used Input(), EventEmitter
RxJS BehaviorSubject, Subject
Manual subscribe()
ChangeDetectionStrategy tricks
This often caused:
Memory leaks
Unnecessary change detection cycles
Complex state logic
Signals solve this by:
Being synchronous
Automatically tracking dependencies
Triggering fine-grained change detection
