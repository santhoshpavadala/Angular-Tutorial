Angular Interview Questions and Answers:

Angular Interview Questions & Answers (5 Years Experience – Angular 13 to 21)

CORE FUNDAMENTALS

1. What is Angular and why is it used?

Angular is a TypeScript-based front-end framework for building scalable Single Page Applications (SPAs). It provides built-in solutions for routing, forms, HTTP communication, dependency injection, and state management. It is suitable for enterprise applications because of its structure and maintainability.

2. What are the main building blocks of Angular?

  • Components
  • Templates
  • Directives
  • Pipes
  • Services
  • Dependency Injection
  • Routing
  • Modules / Standalone Components

3. What is the difference between Angular 13 and Angular 14+?

Angular 13 removed View Engine and made Ivy default. Angular 14 introduced Standalone Components. Angular 16 introduced Signals. Angular 17 introduced new Control Flow syntax. Angular 18–21 improved zoneless support, hydration, and performance optimizations.

4. What is a component?

A component controls a part of the UI. It consists of: - TypeScript class - HTML template - CSS styles - Metadata decorator (@Component)

5. What are lifecycle hooks?

  • ngOnInit
  • ngOnChanges
  • ngAfterViewInit
  • ngOnDestroy

They allow execution of logic at specific stages of component life.

DEPENDENCY INJECTION

6. What is Dependency Injection?

It is a design pattern where dependencies are injected instead of created manually. It improves testability and maintainability.

7. What does providedIn: 'root' mean?

It makes the service a singleton available throughout the application.

8. Explain hierarchical injectors.

Angular creates injectors at root, module, and component levels. If a dependency is not found locally, Angular searches upward.

CHANGE DETECTION & PERFORMANCE

9. How does Angular change detection work?

Angular runs a top-down check of components to detect data changes. Zone.js triggers detection when async tasks complete.

10. What is OnPush strategy?

OnPush runs change detection only when: - Input reference changes - Event occurs - Observable emits - markForCheck() is called

11. Why is OnPush better for large apps?

It reduces unnecessary DOM checks and improves performance.

12. What are common performance mistakes?

  • Not using trackBy
  • Mutating objects instead of replacing
  • Too many subscriptions
  • Heavy logic in templates

SIGNALS (Angular 16+)

13. What are Signals?

Signals are reactive primitives for state management. They provide fine-grained reactivity.

14. What is computed()?

It creates a derived signal from other signals.

15. What is effect()?

Runs side-effects when a signal changes.

16. Signals vs Observables?

Signals are synchronous and good for local UI state. Observables are asynchronous and good for HTTP and streams.

RXJS

17. What is Observable?

An Observable represents a stream of data over time.

18. Subject vs BehaviorSubject?

BehaviorSubject holds the latest value. Subject does not.

19. switchMap vs mergeMap?

switchMap cancels previous request. mergeMap runs parallel requests.

20. How do you prevent memory leaks?

Use takeUntil, async pipe, or takeUntilDestroyed() (Angular 16+).

FORMS

21. Template vs Reactive Forms?

Reactive forms are scalable and testable. Template forms are simple and suitable for small forms.

22. What is FormArray?

Used for dynamic forms where controls are added/removed dynamically.

23. How to create custom validator?

Create a function returning ValidationErrors or null.

ROUTING

24. What are route guards?

Guards control navigation. Examples: CanActivate, CanDeactivate, CanLoad, CanMatch.

25. What is lazy loading?

Loading modules only when needed to reduce initial bundle size.

26. What is PreloadAllModules?

Preloads lazy modules after initial load.

HTTP & INTERCEPTORS

27. What is HttpInterceptor?

Intercepts HTTP requests globally. Used for JWT token, logging, error handling.

28. How to handle global errors?

Use catchError in interceptor.

29. How to cancel HTTP requests?

Use switchMap or AbortController.

STATE MANAGEMENT

30. When to use NgRx?

When application state becomes complex and shared across many modules.

31. What is Component Store?

Lightweight state management for local feature state.

32. What is optimistic update?

Update UI before API confirmation and rollback on failure.

SSR & HYDRATION

33. What is Angular Universal?

Server Side Rendering solution for SEO and performance.

34. What is hydration?

Reusing server-rendered DOM on client side without re-rendering.

SECURITY

35. How does Angular prevent XSS?

Angular sanitizes HTML automatically.

36. What is DomSanitizer?

Used to bypass security when rendering trusted HTML.

ADVANCED ARCHITECTURE

37. How to structure enterprise Angular project?

Feature-based folders, shared components, core services, lazy routes.

38. Smart vs Dumb components?

Smart handles logic, Dumb handles UI only.

39. What is Micro Frontend?

Splitting large frontend into independent deployable apps.

40. How to reduce bundle size?

  • Lazy loading
  • Standalone components
  • Tree shaking
  • Remove unused libraries

TESTING

41. How to test services?

Use TestBed and HttpTestingController.

42. What is mocking?

Replacing real dependency with fake implementation in testing.

Angular Interview Questions & Answers (Basic to Advanced - Angular 13 to 21)

1. What is Angular?

Angular is a TypeScript-based front-end framework used to build scalable, enterprise-level Single Page Applications (SPAs). It follows a component-based architecture and provides built-in solutions for routing, HTTP, forms, dependency injection, and state management.

2. What are the main building blocks of Angular?

  • Components
  • Modules / Standalone Components
  • Templates
  • Directives
  • Pipes
  • Services
  • Dependency Injection
  • Routing

3. What is the difference between Angular 13 and Angular 14+?

Angular 13: Ivy fully enabled, View Engine removed.

Angular 14: Introduced Standalone Components.

Angular 15: Stable standalone APIs.

Angular 16: Introduced Signals.

Angular 17: Control Flow syntax (if, for).

Angular 18-21: Improved zoneless support, performance optimization, signal-based reactivity enhancements.

4. What are Standalone Components?

Standalone components remove the need for NgModules. They simplify architecture and improve tree-shaking.

5. What is Dependency Injection?

Dependency Injection (DI) is a design pattern where dependencies are injected into a class instead of being created manually. It promotes loose coupling and testability.

6. Explain providedIn: 'root'

When a service is provided in 'root', Angular creates a singleton instance available throughout the application.

7. What is Change Detection?

Change Detection is the mechanism by which Angular updates the DOM when component data changes.

Types:

  • Default
  • OnPush

8. What is OnPush Strategy?

OnPush triggers change detection only when:

  • @Input reference changes
  • Observable emits
  • Manual markForCheck()
  • Event triggered

It improves performance in large applications.

9. What is Zone.js?

Zone.js tracks asynchronous operations (like setTimeout, HTTP calls, promises) and triggers change detection automatically.

10. What is Zoneless Angular (Angular 16+)?

Zoneless Angular removes dependency on Zone.js and uses Signals for fine-grained reactivity. It improves performance and gives developers more control.

11. What are Signals (Angular 16+)?

Signals are reactive primitives used for managing state. They provide fine-grained reactivity without full component re-rendering. 

count = signal(0);
count.update(v => v + 1);

12. Difference between Signals and Observables?

Signals Observables
Synchronous Asynchronous
No subscription required Requires subscription
Local state management Async data streams

13. What is RxJS?

RxJS is a library for reactive programming using Observables to handle asynchronous data streams.

14. Explain Subject vs BehaviorSubject vs ReplaySubject

  • Subject: No initial value
  • BehaviorSubject: Holds latest value
  • ReplaySubject: Replays previous values

15. What is switchMap?

switchMap cancels the previous observable when a new value is emitted. Used in search and auto-suggestion scenarios.

16. What is Lazy Loading?

Lazy loading loads modules only when needed, reducing initial bundle size and improving performance.

17. Difference between Template-driven and Reactive Forms?

Template Driven Reactive Forms
Simple Scalable
Less control More control
HTML-driven TypeScript-driven

18. What is a Route Guard?

Route Guards control access to routes. Types: CanActivate, CanDeactivate, CanLoad, CanMatch.

19. What is HttpInterceptor?

Intercepts HTTP requests and responses globally. Used for authentication tokens, logging, and error handling.

20. How to optimize Angular application?

  • Use OnPush
  • Use trackBy in *ngFor
  • Lazy load modules
  • Use standalone components
  • Minimize subscriptions
  • Use Signals

21. What is Angular Universal?

Angular Universal enables Server Side Rendering (SSR) to improve SEO and initial load performance.

22. What is State Management?

State management handles application data flow. Options:

  • Services with BehaviorSubject
  • Signals
  • NgRx
  • Component Store

23. What are Smart and Dumb Components?

  • Smart Components: Handle business logic and API calls
  • Dumb Components: Only UI and Input/Output bindings

24. How do you handle memory leaks?

  • takeUntil
  • Async Pipe
  • DestroyRef (Angular 16+)
  • Unsubscribe in ngOnDestroy

25. What are Angular lifecycle hooks?

  • ngOnInit
  • ngOnChanges
  • ngAfterViewInit
  • ngOnDestroy

26. What is trackBy in *ngFor?

trackBy improves performance by tracking items uniquely instead of recreating DOM elements.

27. Explain new Control Flow syntax (Angular 17+)

                if (condition)

                for (item of items; track item.id)

Improves readability and performance over traditional *ngIf and *ngFor.

28. What is Web Vitals and why important?

Web Vitals measure performance metrics like LCP, CLS, and FID. Important for performance and SEO.

29. What is Micro Frontend Architecture?

Breaking large applications into smaller independently deployable frontend modules. Useful in enterprise projects.

30. How would you design a scalable Angular application?

  • Feature-based folder structure
  • Core & Shared modules
  • Lazy loading
  • State separation
  • Reusable UI components
  • Strict TypeScript rules

31. What are the major improvements in Angular 20 & 21?

  • Enhanced signal-based reactivity
  • Improved zoneless performance
  • Better hydration support
  • Optimized compiler performance
  • Faster build times
