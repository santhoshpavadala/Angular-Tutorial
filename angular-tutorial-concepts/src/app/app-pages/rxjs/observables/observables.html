Obeservable in Angular

What is an Observable?

An Observable is a data stream that emits values over time (asynchronously), and you can listen to those values using subscribe().
Angular uses Observables from RxJS.
Observable = “Data coming continuously over time”

Observables are a core part of RxJS (Reactive Extensions for JavaScript), which Angular uses extensively under the hood.
Think of an Observable as:
A stream of data that arrives over time
Something you can subscribe to
An elegant way to handle asynchronous operations (HTTP calls, user events, WebSockets, etc.)

What Is Observables

An Observable is a data stream that emits values over time.
You can subscribe to it to receive data, errors, or completion notifications.

Where to Use Observables in Angular

1. API Calls (MOST COMMON): Use case: Fetch data from backend return this.http.get('/api/users');

2. Component ↔ Component Communication: Use case: Share data between components

3. Forms (Realtime Input Handling): this.form.valueChanges.subscribe: Live validation, Search boxes, Auto-save forms

4. Search with Debounce

5. Routing Events

6. Real-Time Data (Live Updates): interval(1000).subscribe

7. Async Pipe in Templates (BEST PRACTICE)

8. Error Handling & Data Transformation

9. WebSocket / Live APIs
