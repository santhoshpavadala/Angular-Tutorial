Subject

What is a Subject?

In Angular, a Subject (from RxJS) is a special type of Observable that:

  • ✔ Can emit values (like Observable)
  • ✔ Can be subscribed to (like Observable)
  • ✔ Acts as both Observer + Observable

A Subject is like a bridge or event bus that allows data to be shared between components/services.

    A Subject is:
  • An Observable
  • An Observer
    That means:
  • It can subscribe to other observables.
  • It can emit values manually using next().

Think of it like a data broadcaster 📡 — when it emits, all subscribers receive the value.

Why do we use Subjects?

Normally:
Observables → only emit data
Subjects → emit + listen + share data

Used when:

  • You want real-time communication
  • Share data between components
  • Trigger events manually

Types of Subjects:

  • Subject (Basic)
  • BehaviorSubject ⭐ (Most Important)
  • ReplaySubject
  • AsyncSubject

Subject (Basic)

  • Does NOT store previous value
  • Only emits to current subscribers

BehaviorSubject ⭐ (Most Important)

  • Stores latest value
  • New subscribers get last emitted value

ReplaySubject

  • Stores multiple previous values

AsyncSubject

  • Emits only last value when completed
