Rxjs-Operators

Opearors:

RxJS operators are functions that transform, filter, combine, or control Observables.

Types of Opearors:

  • 1. Creation Operators
  • 2. Pipeable / Transformation Operators
  • 3. Filtering Operators
  • 4. Combination Operators
  • 5. Error Handling Operators
  • 6. Utility / Multicasting Operators

Creation Operators:

  • of()
  • from()
  • interval()
  • timer()
  • range()
  • fromEvent()

Transformation Operators:

  • map()
  • mergeMap() (flatMap)
  • switchMap()
  • concatMap()
  • exhaustMap()

Filtering Operators:

  • filter()
  • take()
  • takeUntil()
  • debounceTime()
  • distinctUntilChanged()

ForkJoin Operator

forkJoin combines multiple observables and waits until ALL complete.
It emits only once when all observables complete.

    Important Notes About forkJoin
  • All observables must COMPLETE
  • If one API fails → entire forkJoin fails
  • Works perfectly for HTTP (because HTTP completes automatically)

SwitchMap

switchMap is a higher-order RxJS operator that maps a value to a new Observable and automatically cancels the previous Observable when a new value arrives.

    Simple meaning:
  • When a new request comes
  • It cancels the previous request
  • And switches to the latest one

That’s why it’s called switchMap.

Why Do We Even Need switchMap?

    Because sometimes:
  • You trigger an API call based on user action
  • User triggers again before first API finishes
  • You don't want old response anymore
  • Without switchMap → both responses come
  • With switchMap → only latest response is kept

MergeMap Operator

mergeMap maps each emitted value to an inner Observable and subscribes to all of them simultaneously without cancelling previous ones.

  • Every input triggers an API
  • All API calls run
  • All responses are processed

MergeMap Examples

concatMap

concatMap maps each emitted value to an inner Observable and executes them sequentially, waiting for each one to complete before starting the next.

  • One request at a time
  • Next starts only after previous completes
  • Maintains order

ConcatMap Examples

exhaustMap

exhaustMap maps each emitted value to an inner Observable, but ignores new emissions while the current inner Observable is still running.

    Simple meaning:
  • If one request is running
  • Ignore new triggers
  • Until current completes
  • It “exhausts” the current observable before accepting new ones.

Easy Mental Model: Imagine You click a Submit button.
While submission is in progress:
Further clicks are ignored
Until request completes.
That’s exhaustMap.
