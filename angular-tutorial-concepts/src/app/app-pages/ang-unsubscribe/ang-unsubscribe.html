Memory Leak in Angular

A memory leak happens when the application keeps objects in memory even after they are no longer needed.

A memory leak occurs when subscriptions or objects remain in memory after the component is destroyed, causing increased memory usage.

In Angular, this usually happens when:

  • You subscribe to an Observable
  • But do not unsubscribe
  • Even after the component is destroyed
  • So the subscription keeps running in memory.

Why it is a problem

    If many subscriptions remain active:
  • Memory usage increases
  • App becomes slow
  • Unexpected API calls or events continue

unsubscribe() in Angular

unsubscribe() is used to stop an Observable subscription when it is no longer needed.
This prevents memory leaks.

Why do we use unsubscribe()?

To stop Observable execution and prevent memory leaks when a component is destroyed.

OnDestroy Lifecycle Hook

OnDestroy is an Angular lifecycle hook that runs just before the component is destroyed.

    It is mainly used for:
  • Cleaning subscriptions
  • Removing event listeners
  • Clearing timers
  • Avoiding memory leaks

What is OnDestroy?

OnDestroy is a lifecycle hook executed before Angular destroys a component, used to clean resources like subscriptions.
