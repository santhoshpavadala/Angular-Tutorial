Decerators

What is a Decorator in Angular?

A Decorator is a special TypeScript function that adds metadata to a class, method, property, or parameter.
In Angular, decorators tell Angular how a class or property should behave.

A decorator is a function that adds metadata to a class, property, method, or parameter so Angular knows how to process and use it.

    Angular uses decorators to define things like:
  • Components
  • Directives
  • Services
  • Modules
  • Input/Output bindings
  • Dependency Injection

Types of Angular Decorators

Angular decorators are mainly 4 types:

  • Class Decorators: Applied to classes: @Component, @Directive, @Pipe, @Injectable, @NgModule
  • Property Decorators: Applied to class properties: @Input, @Output, @ViewChild
  • Method Decorators: Applied to methods: @HostListener
  • Parameter Decorators: Applied to constructor parameters: @Inject, @Optional

Property Decorators: These decorators are applied to class properties.

@Input Decorators:

@Input allows data from parent component to child component.

@Output Decorators:

@Output is used to send data from child component to parent component.

@ViewChild

Used to access a child component or DOM element from the parent component.

View Child Decerator

@ViewChild is used to get a reference of a single child element, directive, or component from the template.

It returns only the first matching element.

@ViewChild is a decorator used to access a single DOM element, directive, or child component from the parent component's template.
