Angular Change Detection

What is Chnage Detection?

Change Detection is the process Angular uses to detect data changes and update the DOM automatically.
Angular continuously checks if component data has changed and updates the UI.

Change Detection checks whether component data has changed and updates the UI accordingly.

2. How Angular Change Detection Works

Angular uses Zone.js to detect async events.

    Zone.js tracks things like:
  • Click events
  • HTTP requests
  • Timers (setTimeout)
  • Promises
  • Observables

Whenever one of these happens, Angular triggers a Change Detection Cycle.

Types of Change Detection Strategies

Angular provides 2 strategies.

1. Default Change Detection

This is the default behavior. Angular checks every component whenever any event occurs.

2. OnPush Change Detection

More optimized strategy.
Angular checks component only when specific conditions occur.

changeDetection: ChangeDetectionStrategy.OnPush

Now Angular will NOT check this component on every cycle.

OnPush triggers change detection only when:

  • Input reference changes
  • Event occurs in component
  • Observable emits value
  • Manual detection

Manual Change Detection

Angular provides ChangeDetectorRef.

constructor(private cd: ChangeDetectorRef)

detectChanges()

this.cd.detectChanges();

"Why does OnPush still update UI when clicking button?"

Even with ChangeDetectionStrategy.OnPush, Angular triggers change detection when events originate from the component such as click events. Therefore when the button click calls nameChange() and updates the value, Angular detects the change and updates the UI.

Button Click, Input Property Change, Observable / HTTP, Manual markForCheck, Reference change then onPush wont works
