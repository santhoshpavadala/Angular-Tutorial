var vs let vs const

"var is function-scoped and allows re-declaration, while let and const are block-scoped. const cannot be reassigned, and let is used when values change. Also, let and const are hoisted but stay in temporal dead zone."

var leaks outside blocks, while let and const stay inside the block

const stops reassigning, but object properties inside a const object can still change

var is function-scoped and allows re-declaration and is initialized as undefined during hoisting.
let and const are block-scoped, cannot be re-declared, and are hoisted but stay in the Temporal Dead Zone.
const cannot be reassigned after initialization.

What are scopes in JavaScript? (global, function, block)

Scope in JavaScript defines the accessibility of variables.
There are three types: global scope, function scope, and block scope.
Global variables are accessible everywhere,
function-scoped variables are accessible only inside the function,
and block-scoped variables (declared with let and const) are accessible only within the block.

Difference Between null and undefined

undefined means a variable has been declared but not assigned a value.
null is an assignment value that represents an intentional absence of value.

What is Hoisting?

Hoisting is JavaScript's behavior where variable and function declarations are moved to the top of their scope during the compilation phase before code execution.

Only declarations are hoisted, not initializations.
Happens during the memory creation phase of the Execution Context.

Variables/functions moved to top of scope. Before initilization we can access variables declared with var and normal functions we can invoke.
let and const and arrow functions doesnot supports Hoistings.

Hoisting is a JavaScript behavior where variable and function declarations are moved to the top of their scope during the compilation phase before execution. Variables declared with var are initialized with undefined, while let and const remain in the Temporal Dead Zone until initialized.

How JavaScript Executes Code (Important for Hoisting)?

JavaScript execution happens in two phases:

1. Memory Creation Phase

JavaScript scans the code and allocates memory.

  • Variables → initialized as undefined
  • Functions → stored completely in memory

2. Execution Phase

Code runs line by line.

The time between variable declaration and initialization where the variable cannot be accessed.

What is a pure function? Pure and Impure functions in javascript

Pure Functions

A pure function always returns the same output for the same input and does not cause any side effects.

  • It has no side effects
  • It is predictable

A Pure Function is a function that:

  • Always returns the same output for the same input
  • Does not modify anything outside the function (no side effects)

A pure function is a function that always returns the same output for the same input and does not modify any external state or variables.

Impure Functions

An impure function may return different outputs for the same input or modify external state (side effects).

An Impure Function is a function that:

  • Produces different outputs for the same input OR
  • Modifies external variables or state

If a function depends on or changes outside data, it is an impure function.

What is a Callback Function?

A callback function is a function that is passed as an argument to another function and is executed later.

In simple words: A callback is a function that runs after another function finishes its task.

A callback function is a function passed as an argument to another function and executed after the main function finishes its task.

Why are callbacks used?

  • Asynchronous operations
  • Event handling
  • Array methods
  • Handling API responses

What is a Promise in JavaScript?

A Promise is an object that represents the eventual completion or failure of an asynchronous operation.

A Promise represents a value that may be available now, later, or never.

Fetching data from an API takes time. A Promise handles that asynchronous result.

Why Promises Are Needed

Before Promises, JavaScript used callbacks, which often led to callback hell.

Why Promises Are Needed

States of a Promise

  • Pending: Initial state (operation not finished)
  • Fulfilled: Operation completed successfully
  • Rejected: Operation failed

Difference between Local and Session storage

What is Web Storage?

Web Storage API allows browsers to store data on the client side.

    There are two types:
  • LocalStorage
  • SessionStorage

Both store data as key-value pairs.

What is LocalStorage?

LocalStorage stores data permanently in the browser until it is manually deleted.
Even if the browser is closed and reopened, the data remains.

localStorage.setItem("username", "Santhosh");
localStorage.setItem("username2", "Santhosh");
localStorage.setItem("username3", "Santhosh");
localStorage.removeItem("username2");
// localStorage.clear();
let name = localStorage.getItem("username");
console.log(name, 'This is Local storage name');

What is SessionStorage?

SessionStorage stores data only for the current browser tab session.
When the tab is closed, the data is automatically deleted.

sessionStorage.setItem("token", "12345");

let token = sessionStorage.getItem("token"); console.log(token);

Event loop in JavaScript

The Event Loop is a mechanism in JavaScript that continuously checks the call stack and moves tasks from the microtask queue or callback queue to the call stack when it becomes empty, allowing JavaScript to handle asynchronous operations.

JavaScript is Single Threaded

JavaScript can execute only one task at a time.
It has one main thread called the Call Stack.

Components of JavaScript Runtime

    To understand the Event Loop, we must know these parts:
  • Call Stack
  • Web APIs
  • Callback Queue (Task Queue)
  • Microtask Queue
  • Event Loop

Call Stack

The Call Stack is where JavaScript executes functions.

Web APIs

Browsers provide Web APIs for asynchronous operations.

Examples: Examples:setTimeout,fetch,DOM events,geolocation

These run outside the call stack.

Callback Queue (Task Queue)

When async operations finish, their callbacks go into the Callback Queue.

setTimeout → finishes → callback queue

Microtask Queue

Some async tasks go into Microtask Queue.

Examples: Promises (.then), MutationObserver, queueMicrotask

Microtasks have higher priority than callback queue.

Event Loop

The Event Loop constantly checks:

Is the Call Stack empty?

If yes:

Take task from queue → push to call stack
